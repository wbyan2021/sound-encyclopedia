# 声音大百科 · 开发文档 v3.0

> 本文档是项目的**固定技术设定**——定义架构、规范、工具链和开发流程，不随产品迭代频繁变动。
> 产品战略和迭代路线图见 [方案.md](方案.md)。
> 核心设计原则：**数据与代码分离、加声音零改代码、社区共建、零成本部署。**

## 一、项目概述

### 1.1 定位

| 维度 | 内容 |
|------|------|
| 产品名称 | 声音大百科（Sound Encyclopedia） |
| 目标用户 | 1-6 岁儿童（家长陪同操作） |
| 核心价值 | 用真实声音帮助孩子认识自然、动物与世界 |
| 商业模式 | 不考虑商业化，专注开源社区共建 |
| 仓库地址 | https://github.com/wbyan2021/sound-encyclopedia |

### 为什么用单仓库？

- 数据和前端放一起，PR 一次提交就能同时加声音和验证效果
- GitHub Pages 部署简单，一个仓库一个地址
- 未来如果声音库长大了，把 `data/` 拆成独立仓库零成本

### 3.2 单条声音 meta.json（每个声音一个）

放在 `data/sounds/{category}/{name}/meta.json`：

```json
{
  "id": "animals.dog",
  "category": "animals",
  "subcategory": "pet",
  "name_zh": "狗",
  "name_en": "Dog",
  "emoji": "🐶",
  "description": "人类最忠实的朋友，喜欢汪汪叫",
  "sounds": [
    { "file": "audio/dog1a.mp3" },
    { "file": "audio/dog1b.mp3" }
  ],
  "tags": ["宠物", "汪汪", "小狗"],
  "license": "CC0-1.0",
  "source": "https://freesound.org/xxx",
  "contributor": "wbyan2021",
  "added_at": "2026-06-17"
}
```

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 格式 `{category}.{name}`，全局唯一 |
| `category` | 是 | 对应 categories.json 中的分类 id |
| `subcategory` | 否 | 子分类 id |
| `name_zh` | 是 | 中文名 |
| `name_en` | 是 | 英文名（也是目录名） |
| `emoji` | 是 | 前端显示用（暂用 emoji，未来可换图片） |
| `description` | 否 | 一句话描述 |
| `sounds` | 是 | 音频文件数组，至少 1 条 |
| `sounds[].file` | 是 | 相对 meta.json 的路径 |
| `fun_fact` | 否 | 儿童友好科普文案（30-50字），播放后底部自动展示。可用 AI 批量生成，人工审核后合并 |
| `fun_fact_ai_generated` | 否 | `true` 表示 fun_fact 由 AI 生成。前端展示时添加「🤖」标记，保持透明度 |
| `tts.name_zh` | 否 | AI 朗读中文名音频路径（相对 meta.json），如 `generated/name-zh.mp3` |
| `tts.name_en` | 否 | AI 朗读英文名音频路径 |
| `tts.fun_fact` | 否 | AI 朗读科普文案音频路径 |
| `tags` | 否 | 搜索标签，支持中文名、英文名、拼音多维检索 |
| `license` | 是 | 必须是 CC0 或可商用授权 |
| `source` | 是 | 音频来源 URL |
| `contributor` | 是 | GitHub 用户名 |
| `added_at` | 是 | 添加日期 |

## 四、构建脚本

### 4.1 build-manifest.js

**职责**：扫描 `data/sounds/` 下所有 meta.json + 读取 categories.json → 生成 manifest.json

**运行方式**：
```bash
node scripts/build-manifest.js              # 生成 manifest.json
node scripts/build-manifest.js --dry-run    # 只输出统计不写文件
```

**版本号逻辑**：
- 格式：`YYYY.MM.DD.N`
- 读取已有 manifest.json（如存在），同日则 N+1，不同日期则 N=1

**输出示例**：
```
===== 构建统计 =====
  版本:            2026.06.24.2
  分类数:          4
  声音条目数:      38
  音频文件总数:    96
====================
✓ manifest.json 已生成
```

### 4.2 validate.js

**职责**：校验 data/sounds/ 下的所有数据

**校验规则**：

| 检查项 | 规则 |
|--------|------|
| meta.json 必填字段 | id, category, name_zh, name_en, emoji, sounds, license, source, contributor |
| id 格式 | 必须是 `{category}.{name}` 格式 |
| category 有效性 | 必须存在于 categories.json 中 |
| id 唯一性 | 不能有重复的 id |
| 音频文件存在 | sounds[].file 指向的文件必须实际存在 |
| 音频格式 | 必须是 .mp3 扩展名 |
| 音频大小 | < 300KB（警告，不阻断） |
| 目录命名 | 只能用小写字母和下划线 |

**运行方式**：
```bash
node scripts/validate.js                                    # 校验所有
node scripts/validate.js --path data/sounds/animals/dog     # 只校验指定路径
```

### 4.3 ai-generate-fun-fact.js

**职责**：批量调用 MiniMax LLM，为声音生成儿童友好科普文案（`fun_fact`）。

**运行方式**：
```bash
node scripts/ai-generate-fun-fact.js                    # 为所有缺少 fun_fact 的声音生成
node scripts/ai-generate-fun-fact.js --force            # 强制重新生成全部
node scripts/ai-generate-fun-fact.js --category animals # 只生成动物分类
node scripts/ai-generate-fun-fact.js --id animals.dog   # 只生成指定声音
```

**环境要求**：需要在 `.env` 中配置 `MINIMAX_API_KEY`。

**输出**：直接写入对应声音的 `meta.json`，并标记 `fun_fact_ai_generated: true`。

### 4.4 ai-generate-tts.js

**职责**：批量调用 MiniMax TTS API，为中文名、英文名、科普文案生成 AI 朗读音频。

**运行方式**：
```bash
node scripts/ai-generate-tts.js                         # 为所有缺少 TTS 的声音生成
node scripts/ai-generate-tts.js --category animals      # 只生成动物分类
node scripts/ai-generate-tts.js --id animals.dog        # 只生成指定声音
```

**输出**：音频文件保存到 `data/sounds/{category}/{name}/generated/` 目录。

### 4.5 admin-server.js（本地管理后台）

**职责**：提供 Web 界面，方便可视化管理 AI 生成和数据统计。

**启动方式**：
```bash
node scripts/admin-server.js
# 浏览器打开 http://localhost:3099
```

**功能**：
- 📊 数据统计仪表盘（声音总数、AI 文案数、TTS 数、分类计数）
- 🔑 API Key 配置管理（保存到 `.env`）
- 🤖 一键批量生成科普文案
- 🔊 一键批量生成 TTS 朗读
- 📦 一键重建 manifest
- 🚀 一键全流程（文案 + TTS + 构建）
- 🖥️ 实时终端输出（SSE 流式）
- 安全：仅绑定 `127.0.0.1`，不对外暴露

## 五、GitHub 自动化

### 5.1 PR 校验（validate-pr.yml）

**触发**：每次 Pull Request 修改 data/sounds/ 时

**流程**：
```
PR 提交 → GitHub Actions → node scripts/validate.js → 通过/失败
```

校验失败会阻止 PR 合并。

### 5.2 自动构建（build-manifest.yml）

**触发**：PR 合并到 main 分支

**流程**：
```
合并到 main → GitHub Actions → node scripts/build-manifest.js → 自动提交 manifest.json
```

不需要人工跑脚本。

### 5.3 Issue/PR 模板

- **Issue 模板**（sound-request.yml）：普通用户提"想加 XX 声音"需求
- **PR 模板**（PULL_REQUEST_TEMPLATE.md）：贡献者填写声音名称、分类、版权确认

## 六、前端设计

### 6.1 技术栈

| 项 | 选择 | 理由 |
|----|------|------|
| 框架 | 原生 HTML/CSS/JS | 简单、零依赖、易扩展 |
| 设计语言 | Glassmorphism + 动态 CSS 变量 | 现代扁平拟物风格，沉浸交互体验 |
| 字体库 | 系统字体栈（Apple/Windows/Linux） | 零依赖、加载零延迟，国内网络友好 |
| 数据 | fetch manifest.json | 数据驱动，加声音不改代码 |
| 音频 | HTML5 Audio | 原生支持 |
| 离线 | Service Worker (sw.js) | PWA，音频 cache-first 策略 |
| 本地缓存 | localStorage | 用于保存收藏列表和已探索进度 |
| 部署 | GitHub Pages | 免费、自动 |

### 6.2 核心交互

- **开始页**：Canvas 动态粒子背景 + Glassmorphism 毛玻璃卡片 + 标题动画
- **懒加载**：点击时加载音频，加载过的自动缓存，不再全量预加载
- **分类 tabs**：从 manifest.categories 动态生成，只显示有数据的分类
- **子分类导航**：选大分类后显示对应的子分类 tab
- **声音卡片**：emoji + 名称 + 收藏按钮，点击触发播放 + 动画 + 底部科普面板
- **搜索框**：支持中文名、英文名、标签实时检索
- **收藏系统**：点击心形按钮收藏，专属"我的收藏"分类页，localStorage 持久化
- **探索进度系统**：记录已播放声音，显示百分比进度，解锁勋章称号（见习生→探险队长→大师→大冒险家）
- **随机探索**：随机跳转到一个声音并播放
- **底部科普面板（Bottom Sheet）**：播放后底部滑出，展示 emoji + 名称 + fun_fact 科普文案 + 标签 + 贡献者
- **AI 语音朗读（TTS）**：底部面板内三个独立按钮——读中文名 / 读英文名 / 听 AI 讲解，点击即播，清晰标注「🤖 AI 朗读」
- **动态配色**：切换分类时页面背景和主色调自动过渡切换
- **音量控制**：右下角悬浮滑块，点击图标可静音
- **动画反馈**：弹跳旋转 + 水波纹扩散 + 星星飞溅
- **多音频随机轮播**：拥有多条真实音频的条目，每次点击随机抽取播放

### 6.3 数据流

```
页面加载 → fetch('data/manifest.json') → 解析 JSON
  → 渲染分类 tabs（从 manifest.categories）
  → 渲染声音卡片（从 manifest.sounds）
  → 点击卡片 → 随机播放该声音的音频文件
```

## 七、离线方案

| 内容类型 | 缓存策略 | 说明 |
|----------|---------|------|
| manifest.json | network-first | 保证新声音能上线 |
| 音频文件 (.mp3) | cache-first | 省流量，加载过就缓存 |
| index.html / sw.js | cache-first | 离线可用 |

> ✅ 已修正：manifest.json 为 network-first，mp3 为 cache-first，index.html/sw.js 为 network-first。

## 八、音频规范

| 项 | 标准 |
|----|------|
| 格式 | MP3 |
| 采样率 | 44100 Hz |
| 比特率 | 128 kbps |
| 时长 | 1-10 秒（自然音可到 30 秒） |
| 响度 | -16 LUFS（归一化，避免忽大忽小） |
| 体积 | 单条 < 300KB |

## 十、贡献流程

```
打开 GitHub Issues → 选"声音需求"模板 → 描述想要的声音 → 等维护者添加
```

```bash
git clone https://github.com/wbyan2021/sound-encyclopedia.git
cd sound-encyclopedia

# 1. 创建声音目录
mkdir -p data/sounds/nature/rain/audio

# 2. 放入音频文件
cp rain.mp3 data/sounds/nature/rain/audio/

# 3. 创建 meta.json（参考第三章）

# 4. 本地校验
node scripts/validate.js

# 5. 生成索引（CI 会自动做，本地可选）
node scripts/build-manifest.js

# 6. 提交 PR
git checkout -b add-rain
git add .
git commit -m "add: rain sound"
git push origin add-rain
```

## 十二、关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 仓库结构 | 单仓库（data + 前端） | PR 一次搞定，未来可拆 |
| 图片方案 | emoji | 零版权、加载快、儿童友好 |
| 音频格式 | MP3 | 兼容性最好 |
| 部署方式 | GitHub Pages | 免费、自动 |
| 管理方式 | Git + 脚本 + 本地管理后台 | 降低维护成本，本地开发用 admin-server.js |
| AI 工具 | scripts/lib/minimax.js（LLM + TTS） | CommonJS，脚本层统一引用 |
| AI 服务 | MiniMax | 中文效果好、TTS 自然、价格实惠 |
| AI 内容透明度 | 所有 AI 生成/朗读内容标注「🤖」 | 对家长透明，建立信任 |
| TTS 音频存放 | generated/ 目录，与 audio/ 分离 | 避免真实录音与 AI 朗读混淆 |
| 商业模式 | 不考虑 | 专注开源社区共建 |
| 版权协议 | CC0（素材）+ MIT（代码） | 最干净 |
